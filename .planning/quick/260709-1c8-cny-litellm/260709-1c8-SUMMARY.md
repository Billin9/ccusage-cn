# Quick Task 260709-1c8: 中国模型人民币直接定价

为中国模型（DeepSeek V4 Pro/Flash, GLM-5.2, Kimi K2.6, Qwen3-Max）实现官方人民币直接定价，消除 LiteLLM 网络依赖和 USD->CNY 双重转换误差。

## 关键决策

1. **cnPricing 惰性加载**: `--help` 模式下完全跳过 cnPricing 加载，避免 L1 fetch 5s 超时延迟。非帮助模式通过 `Promise.all` 与汇率加载并发执行。
2. **流式模式限制**: `createTextTransform` 仅接受 cnPricing 参数用于签名一致性，实际无法使用（逐块输出无结构化数据）。已在 JSDoc 中明确记录此限制。
3. **文本覆盖策略**: 按 `│` 列分割解析模型名称和 token 计数，而非正则匹配。更健壮地处理 ANSI 码和上游输出格式变化。
4. **JSON 覆盖时机**: 在 `addCostCNY`（USD*汇率）之后执行，通过差值调整 `totalCostCNY`，无需修改现有转换管道。

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] cnPricing L1 fetch 5s 超时阻塞 --help 输出**
- **Found during:** Task 2 verification
- **Issue:** `loadCnPricing()` 在 `main()` 顶部被 `await`，其 L1 GitHub fetch 有 5s 超时。导致 `--help` 路径延迟 5s 以上，集成测试超时。
- **Fix:** 将 cnPricing 加载改为惰性 Promise，仅在 `--help` 以外的分支中 `await`。JSON 和缓冲模式通过 `Promise.all` 与汇率加载并发执行。
- **Files modified:** `bin/cli.js`
- **Commit:** 9aa38d1 (集成在 Task 2 提交中)

**2. [Rule 1 - Bug] pricing/cn-models.json 路径计算错误**
- **Found during:** Task 1 验证阶段

- **Fix:** 改为 `../../pricing/cn-models.json`
- **Files modified:** `src/pricing/cn-provider.js`
- **Commit:** d34f79e (集成在 Task 1 提交中)

**3. [Rule 1 - Bug] 文本模式覆盖路径未被触发**

- **Fix:** 在回退分支中缓冲 fallbackTransform 输出，然后调用 `overrideCnTextOutput`。
- **Files modified:** `src/output-transform.js`
- **Commit:** 9aa38d1 (集成在 Task 2 提交中)

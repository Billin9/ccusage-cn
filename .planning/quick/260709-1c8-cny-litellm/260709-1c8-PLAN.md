<objective>
为中国模型（DeepSeek V4 Pro/Flash、GLM-5.2、Kimi K2.6、Qwen3-Max）实现官方人民币直接定价，消除 LiteLLM 网络依赖和 USD->CNY 双重转换误差。

**Purpose:** 当前所有模型的费用都经过 "LiteLLM USD 定价 -> USD 费用 -> 乘以汇率 -> 人民币" 的路径。对于中国模型，这意味着：
1. LiteLLM 将官方 CNY 定价乘以汇率转为 USD（第一层误差）
2. 我们再乘以汇率转回 CNY（第二层误差）
3. 网络请求 LiteLLM 增加了延迟和失败风险

本计划实现官方 CNY 直接定价（per D-01、D-04、D-05、D-06）、三层定价加载（per D-03）和格式规范（per D-02）。

**Output:**
- `pricing/cn-models.json` — 自维护的中国模型官方 CNY 定价文件
- `src/pricing/cn-provider.js` — 三层回退加载定价数据
- `src/pricing/cost-calculator.js` — 中国模型识别 + 直接 CNY 费用计算
- 修改 `src/output-transform.js` — JSON 和文本模式的 cost 覆盖逻辑
- 修改 `bin/cli.js` — 集成 cnPricing 到主流程
- `test/cn-pricing.test.js` — 新模块测试覆盖
</objective>

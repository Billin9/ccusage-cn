    在 src/utils.js 中：
      1. 导出 getCacheDir() 函数：
         - process.platform === 'win32' 时返回 join(process.env.LOCALAPPDATA, 'ccusage-cn', 'cache')
         - 否则返回 join(os.homedir(), '.ccusage-cn', 'cache')
         - 使用 import { homedir } from 'node:os' 和 import { join } from 'node:path'
      2. 导出 formatCNY(amount) 函数：返回 parseFloat(amount.toFixed(2))，保留两位小数精度 per Claude's Discretion

    在 src/exchange-rate.js 中（per D-08）：
      1. import getCacheDir 和 formatCNY 从 utils.js
      2. 定义常量：
         - CACHE_TTL = 24 * 60 * 60 * 1000（24 小时）
         - DEFAULT_RATE = 7.2（硬编码默认值）
         - RATE_API_URL = 'https://cdn.jsdelivr.net/npm/@fawazahmed0/currency-api@latest/v1/currencies/usd.json'
         - FETCH_TIMEOUT = 5000（5 秒，per Claude's Discretion）
      3. 导出 async function getExchangeRate()：
         - 第一层（最高优先级）：读取 process.env.CCUSAGE_CNY_RATE
           - 用正则 /^\d+(\.\d+)?$/ 验证，确保是有效正数
           - 通过则 return parseFloat(envRate)（per D-08 第 1 项）
         - 第二层（CDN 缓存）：
           - 尝试 const cached = await readCache()
           - 如果缓存命中且未过期（Date.now() - cached.timestamp &lt; CACHE_TTL），直接 return cached.rate
           - 如果缓存命中但过期，后台静默刷新：fetchAndCache().catch(() => {})，不阻塞本次输出，return cached.rate
           - 如果缓存未命中，继续第三层
         - 第三层（CDN API fetch）：
           - 执行 fetchWithTimeout(FETCH_TIMEOUT)
           - 成功：await cacheRate(rate)，return rate
           - 失败（网络错误/超时/解析错误）：return DEFAULT_RATE
         - 读取缓存的 readCache()：
           - 用 node:fs/promises readFile + JSON.parse
           - 返回 { rate: number, timestamp: number } 或 null（文件不存在/格式错误/无效时返回 null）
         - CDN fetch 实现 fetchRate()：
           - 用 globalThis.fetch(RATE_API_URL, { signal: AbortSignal.timeout(FETCH_TIMEOUT) })
           - 解析 response.json()，访问 data.usd.cny
         - 写入缓存的 cacheRate(rate)：
           - mkdir(getCacheDir(), { recursive: true })
           - 使用原子写入模式：写临时文件 → writeFile(tmpPath) → rename(tmpPath, cacheFile)
           - 写入格式：JSON.stringify({ rate, timestamp: Date.now() })
           - 静默失败（catch 什么都不做）
  </action>

       - 维护内部 this._remainder 字段处理 chunk 边界不完整 $ 数字（per Pitfall 4 的避免方案）

       - 执行替换（per COST-01）：
         - text.replace(/\$(\d+\.?\d*)/g, (m, amt) => `¥${(parseFloat(amt) * rate).toFixed(2)}`)
         - text.replace(/Cost \(USD\)/g, 'Cost (CNY)')
       - flush 时将 remainder 推入流

    2. 导出 createJsonTransform(rate)：
       - 返回一个 Transform stream 实例
       - 收集所有 chunk 到内部 _chunks 数组

         - 尝试 JSON.parse(raw)

           - this.push(result)
           - 失败：this.push(raw) — 原样透传（per Pitfall 2 的避免方案：决不因解析失败崩溃）

    3. 模块不依赖 exchange-rate.js — 由调用方传入 rate 参数（解耦设计，后续测试可直接传入模拟值）

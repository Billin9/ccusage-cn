      1. import { spawn } from 'node:child_process'，import process from 'node:process'

      2. 导出 createSpawner(binaryPath, args)：
         - 调用 spawn(binaryPath, args, {
             stdio: ['inherit', 'pipe', 'inherit'],  // per D-04
             env: { ...process.env, FORCE_COLOR: '1' }  // per D-07
           })
         - 注册信号转发处理器（per D-05）：
           const onSigint = () => { try { child.kill('SIGINT'); } catch {} };
           const onSigterm = () => { try { child.kill('SIGTERM'); } catch {} };
           process.on('SIGINT', onSigint);
           process.on('SIGTERM', onSigterm);
         - 返回 child 引用（用于后续 pipe stdout）
         - 提供 cleanup 函数：process.off('SIGINT', onSigint); process.off('SIGTERM', onSigterm);

      3. 导出 createExitHandler(child, cleanupFn)：
         - 返回 child.on('exit', (code, signal) => {
             cleanupFn();
             if (signal !== null) {
               // 子进程被信号终止，以相同信号退出（per D-05: process.exit(128 + signalNumber)）
               process.kill(process.pid, signal);
             } else {
               process.exit(code ?? 0);  // 传播退出码（per CLI-02 和 D-10）
             }
           })
         - 关于 Windows 兼容：Windows 不支持 SIGINT，child.kill('SIGINT') 会被忽略或转为 SIGTERM；
           child.kill() 无参时 Windows 行为正确（terminate process）。在 win32 上信号转发仍注册但不报错。
  </action>

    3. 主函数 async function main()：
       a. 获取 CLI 参数：const args = process.argv.slice(2)（per D-09）
       b. 检测模式：const isJson = args.includes('--json')（per D-06）
       c. 解析二进制路径：

       d. 获取汇率：const rate = await getExchangeRate()（非阻塞）
       e. 创建进程：

          // child 是 spawn() 返回的 ChildProcess 对象

            // JSON 模式（per D-06 JSON 路径）：
            // 使用 createJsonTransform(rate)

          } else {
            // 文本模式（per D-06 文本路径）：
            // 使用 createTextTransform(rate)
            child.stdout.pipe(createTextTransform(rate)).pipe(process.stdout)
          }
          // stderr 透传：spawn 时已设为 'inherit'（per D-10）
          // 不需 pipe stderr

    4. ---help 处理（per D-10）：

       - 无需单独的 --help 打印逻辑（上游接管所有帮助输出）

    6. 关于 stdin 透传（非 blocking 需求但建议实现）：
       - 如果用户通过 pipe 输入（如 echo '{}' | ccusage-cn），spawn 时 stdio[0]='inherit'

       - 不需要额外处理

    7. 关于信号处理器：
       - createSpawner 内部已注册 SIGINT/SIGTERM → child.kill()
       - createExitHandler 在子进程退出后 cleanup
       - 注意：process.on('SIGINT') 在子进程退出后需解注册，避免父进程残留 handler

<threat_model>
## 信任边界

| 边界 | 描述 |
|------|------|
| CLI 入口 → 上游二进制 | process.argv.slice(2) 参数通过 spawn 传递给上游二进制 |
| 用户终端信号 → 进程管理 | SIGINT/SIGTERM 信号从 wrapper 转发到子进程 |
| 子进程退出 → Shell | 退出码从子进程通过 wrapper 传播到父进程 |
| 上游 stderr → 用户终端 | stderr 使用 inherit 透传，不经过管道处理 |

## STRIDE 威胁登记表

| 威胁 ID | 类别 | 组件 | 处置 | 缓解方案 |
|---------|------|------|------|----------|
| T-02-01 | Tampering | CLI 参数注入 | mitigate | 使用 spawn(command, args[]) 数组参数形式（per D-04），绝不拼接 shell 字符串，防止参数注入 |
| T-02-02 | Spoofing | 退出码篡改 | mitigate | createExitHandler 从 child.on('exit') 获取真实退出码，通过 process.exit(code) 原样传播（per CLI-02） |
| T-02-03 | Denial of Service | 孤儿进程 | mitigate | 注册 SIGINT/SIGTERM 处理器在 spawn 后立即执行 child.kill()；子进程退出后 cleanup 信号处理器（per D-05） |
| T-02-04 | Tampering | 二进制路径劫持 | mitigate | resolveBinary 使用上游 resolveCliRuntime，路径限制在 node_modules/ccusage 内；遵循 D-03 |
| T-02-05 | Spoofing | 错误的 JSON 转换 | mitigate | createJsonTransform 遇到 JSON 解析失败时原样透传不修改（不吞数据、不产生错误输出） |
| T-02-06 | Information Disclosure | CCUSAGE_CNY_RATE 泄露 | accept | 环境变量只在进程内部读取，不写入日志或输出 |
| T-02-SC | Tampering | 包安装合法性 | mitigate | 在 RESEARCH.md Package Legitimacy Audit 中已验证 ccusage 和 vitest 均为 Approved |
</threat_model>
